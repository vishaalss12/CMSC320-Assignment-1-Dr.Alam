Q3: What is the difference between merging and rebasing?

Answer: The difference between merging and rebasing is that when the git merge command is ran, the version histories from the different branches are combined into a merge commit, that has parent commits from the previously pushed branches. Git rebase moves the sequence of commits from one branch on top of another branch. The advantage of rebasing, is that we have a clear more linear commit history.
